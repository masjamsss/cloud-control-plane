# Primarily for PG-3 (word-bounded, case-insensitive estate vocabulary sourced from the
# untracked denylist — the committed script names no estate word). Word-boundary matching
# already exempts vendor tokens with no boundary inside them — those need NO entry here.
# This file is for the rare case where a genuine word-bounded hit is nonetheless reviewed
# and accepted as safe to ship public: cloud-provider API surface that legitimately
# contains a word the denylist also bans.
#
# Expected to stay near-empty. Every entry needs a human-reviewed reason — this is not a
# bulk-suppress list.
#
# Format: three tab-separated fields per line, blank lines and full-line `#` comments
# ignored:
#
#   <file-path-or-glob>\t<exact-substring-that-must-appear-on-the-flagged-line>\t<reason>
#
# A hit is suppressed only when BOTH the file matches the first field AND the flagged
# line contains the literal substring in the second field — matching is deliberately
# file+substring scoped, never a bare word, so approving one occurrence in one file never
# blanket-exempts the word everywhere else it might leak.
